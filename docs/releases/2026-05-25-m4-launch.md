## 1. 用户面新增功能（M4：在线询价）

之前 M3 联系页只能展示电话/微信，访客自己拨号联系。M4 加了**弹窗式询价表单**。

### 访客侧

- **商品详情页** → 点「询价」→ 弹出 dialog（自动带上商品名）→ 填称呼 / 联系方式 / 留言 → 提交
- **联系页** → 点「提交咨询」→ 弹窗（不带商品）→ 提交通用咨询
- 提交成功显示「已收到」，限流提示「提交过于频繁」
- 中英文双语，按当前 locale 自动切换

### 后台侧

- 新增侧栏入口「**询价**」
- `/admin/inquiries` 列表页：未读 / 已读 / 已回复三态过滤
- 每条询价显示时间、访客信息、商品（点击新窗口打开详情）、留言
- 操作：「标记已读」/「删除」
- 仪表盘的"本周询价数"现在不再是 0 了

### 安全 & 隐私

- 服务端 Zod 严格校验字段长度（name 60 / contact 80 / message 1000）
- **同 IP 5 分钟最多 3 次提交**——超过返回 429 RATE_LIMITED
- 访客的 IP **不入库**——只存 sha256 截 16 hex 作为限流键
- 联系方式（手机号/微信号）明文入库，仅后台可见

### 技术细节

- 公开端点：`POST /api/inquiries`，无需鉴权（中间件白名单）
- 后台端点：`/api/admin/inquiries` 三件套（GET 列表 / PATCH 标已读 / DELETE）
- 限流复用 M1 的 `lib/rate-limit.ts` 的 `inquiryLimiter` 实例（`{ windowMs: 5*60_000, max: 3 }`）
- 测试覆盖：lib 10 个 + API 12 个 + E2E 2 个

## 2. 生产部署（首次上线）

### 部署栈

```
访客

    → Nginx (443, SSL termination)
       → /uploads/      → 直接读 ~/luxury/public/uploads/（alias 直出）
       → /_next/static/ → 直接读 .next/static/
       → /              → proxy_pass http://127.0.0.1:3000 (Next.js / PM2)
       → /tqvS1ra1vBs   → 保留旧 Xray 隐藏路径（如果你还在用代理）
                                    ↓
                              PostgreSQL (Docker, 绑 127.0.0.1:5432)
```

## 3. 部署中的代码修复

部署期间从生产暴露出的 bug，逐一修了。

### 3.1 `fix(api): admin product PATCH accepts empty originalPrice`

**Commit**: `4e17ce4`

后台编辑一件没有「原价」的商品时，表单 input 是空字符串 `""`，服务端 Zod schema 的 `originalPrice: z.string().regex(/^\d+(\.\d{1,2})?$/)` 不接受空——整个保存请求 400。

修：
- Schema 加 `|^$` 允许空字符串
- Handler 把空字符串当 NULL 处理
- 400 响应额外返回 Zod `issues`，方便以后排错

### 3.2 `fix(images): disable next/image optimizer`

**Commit**: `be9796c`

两个问题叠加：

a. **外部 demo 图（loremflickr）的 Node fetch ETIMEDOUT**——服务器 IPv6 配置存在但实际不通；Node 走 Happy Eyeballs 同时拨 IPv4+IPv6，IPv6 拒绝拖死整个 fetch。

b. **后台上传的图 Next.js 404**——`next start` 生产模式不动态扫描 `public/`，上传后的新文件 Next.js 自己看不见，图片优化器从自己服务器拉源图也拉 null。

两个一起修了——直接 `images.unoptimized: true`：

- `/uploads/*` 走浏览器 → Nginx alias 直出（已配）
- 外部 demo 图浏览器直拉源 CDN，不经过我们服务器

代价：失去自动 WebP 转换、srcset 响应式尺寸。当前规模可接受；接 COS+CDN 时撤销这个 flag、配自定义 loader。

### 3.3 `fix: stable orderBy tiebreaker for listInquiriesAdmin`

**Commit**: `e21085f`

后台询价列表 `orderBy: { createdAt: 'desc' }` 没有二级排序键，毫秒级同时入库的多条记录顺序不稳定。改成 `[{ createdAt: 'desc' }, { id: 'desc' }]`。cuid 是时间前缀字典序，做次序兜底很自然。

### 3.4 `fix(api): rate-limit before product lookup`

**Commit**: `bb7f14f`

`POST /api/inquiries` 原先顺序是「Zod parse → product lookup → 限流 → DB 写入」，意味着攻击者可以无限制地用合法格式的 cuid 探测产品存在性（每次走一遍 `prisma.product.findUnique` 但不被限流计数）。

调整为「Zod parse → 限流 → product lookup → DB 写入」，跟 admin login 路由对齐。顺手把 cuid regex 的 `/i` 大小写不敏感 flag 去掉（cuid 本身全小写）。

4. **询价回复闭环**——schema 已留 `REPLIED` 状态字段，UI 暂不支持。M5 可以加邮件/短信通知模板

6. **fail2ban**——挡 SSH 爆破
