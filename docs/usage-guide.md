# 使用手册

面向**店主与助手**——讲清楚怎么把商品和联系方式从后台填进去、最终在公开站长什么样。开发者请看 `README.md` 和 `docs/superpowers/`。

---

## 1. 角色与入口

| 角色 | 入口 | 能干什么 |
|---|---|---|
| 访客（买家） | `/zh`（或 `/en`） | 浏览、筛选、收藏、看联系方式找你 |
| **店主 OWNER** | `/admin/login` | 全部后台权限 + 管理员管理 |
| **助手 ADMIN** | `/admin/login` | 除「管理员管理」外其它都行 |

公开站和后台**域名相同**，靠路径分。访客不会撞见后台——`/admin/**` 必须登录，未登录直接跳转登录页。

---

## 2. 首次登录

1. 浏览器打开 `http://你的域名/admin/login`
2. 邮箱：技术同学初始化时给你的（默认是 `owner@example.com`）
3. 密码：初始化时一次性打印的，按要求**首次登录后立刻去 `/admin/users` 改密码**
4. 登录成功跳到 `/admin` 仪表盘——三块指标：在售数 / 草稿数 / 本周询价数

> **忘记密码？** 没有自助找回。让技术同学跑一行 Node 脚本把密码哈希覆盖掉，然后口头给你新密码（线下渠道传），登录后立即改成自己的。

会话有效期 **7 天**，过期或主动登出后要重新输密码。

## 3. 日常工作流

### 3.1 上一件新商品（最常用）

> **目标**：从"刚收到的二手货"到"访客能看到、能收藏、能询价"。

**步骤**：

1. `/admin/products` → 右上角 **+ 新建商品**
2. 切到「中文」Tab，填**标题 + 描述**
   - 标题示例：`劳力士 潜航者型 126610LN 黑水鬼`
   - 描述：年份、附件齐全度、状况要点、议价空间
3. 切到「英文」Tab，填 **titleEn + descEn**
   - 空着也行——访客切到英文站时会**回退到中文**显示
4. 填**品牌**（自由文本）和**分类**（下拉，没有合适的先去 `/admin/categories` 加）
5. 填**价格**（必填，单位元，整数）和**原价**（可空；填了会在公开页显示划线对比，制造"已优惠"感）
6. 选**成色**（5 档，见下表）
7. 可选：**编号**（手表序列号、包码 etc.）和**尺寸**——尺寸是 key-value，比如：
   - 手表：`diameter: 41mm`
   - 包：`length: 25cm`、`width: 13cm`、`height: 20cm`
8. **上传商品图**（PHOTO）——拖拽进上传区。第一张是封面，列表/首页/收藏页都拿它；详情页轮播按顺序。建议 4-8 张，第一张是正面全景。
9. **上传鉴定附件**（CERT）——证书、原盒发票、保养卡的扫描或拍照。详情页底部"鉴定附件"折叠区会出现；访客点开能放大查看。
10. **状态先留 DRAFT**——拍图、改文案随便折腾，访客都看不见。一切就绪后切到 **AVAILABLE**，公开站立刻出现。

#### 成色含义

| 选项 | 中文 | 用法 |
|---|---|---|
| `NEW` | 全新 | 未拆封、未使用 |
| `LIKE_NEW` | 近全新 | 用过 1-2 次，无肉眼可见使用痕迹 |
| `EXCELLENT` | 极佳 | 轻度使用痕迹，无划痕磕碰 |
| `GOOD` | 良好 | 正常使用痕迹，无明显损伤 |
| `FAIR` | 一般 | 有明显使用痕迹或小瑕疵，已在描述中说明 |

#### 状态生命周期

```
新建 → DRAFT ─上架→ AVAILABLE ─有人定→ RESERVED ─成交→ SOLD
                       │                    │
                       │                    └─谈崩→ 回 AVAILABLE
                       │
                       └─不卖了→ ARCHIVED
```

**只有 `AVAILABLE` 在公开站可见**，其他四种状态访客和搜索引擎都看不到。具体含义：

| 状态 | 含义 | 公开站可见 | 何时用 |
|---|---|---|---|
| DRAFT | 草稿 | ❌ | 信息没填完、图没传齐 |
| **AVAILABLE** | **在售** | ✅ | 正式上架 |
| RESERVED | 已保留 | ❌ | 客户付了定金还没正式交易 |
| SOLD | 已售 | ❌ | 已成交，保留历史记录 |
| ARCHIVED | 归档 | ❌ | 不再卖也不想物理删除 |

### 3.2 调整已上架商品

`/admin/products` 是商品列表。能做的事：

- **直接改状态**：表格里每行的状态下拉，改完即时生效
- **编辑**：点商品标题或「编辑」进编辑页，所有字段都能改（包括加图、删图、拖拽重排）
- **删除**：彻底从 DB 移除，图片记录连带删（图片文件本身不删，仍在 `public/uploads/`，但没有任何引用）
- **筛选**：顶部状态下拉 + 搜索框（标题或编号）

> **建议**：用 `SOLD` 软删除，少用「删除」。SOLD 商品不出现在公开站，但你日后想统计销售情况还能查到。

### 3.3 分类管理

`/admin/categories`：

- **Slug**：URL 用，全小写英文短词（如 `watch`、`bag`、`jewelry`）。一旦商品挂上去，不建议改 slug——会改详情页 URL，可能导致旧链接 404
- **中文名 / 英文名**：分别显示在两个语种的筛选侧栏和首页分类入口
- **排序**：拖拽改 sortOrder，决定公开站分类入口的展示顺序

### 3.4 联系方式（最重要的一项设置）

`/admin/settings` 三个字段，影响整个公开站：

| 字段 | 显示在哪 | 建议 |
|---|---|---|
| `contact_phone` | `/zh/contact` 联系页 — 大字号、可点拨打 | 填**店面或店主直拨号**，包含国家区号，如 `+86 138 0000 0000` |
| `contact_wechat_id` | `/zh/contact` 联系页 — 微信号文本 | 微信号，让访客好"添加朋友" |
| `contact_wechat_qr_url` | `/zh/contact` 联系页 — 二维码图片 | 截一张你的微信二维码上传到 `/uploads/`，把路径粘进来；留空就不显示 |
| `brand_options` | 暂未在界面使用 | 留空即可 |

> 二维码图怎么传：随便去 `/admin/products/new` 用「上传商品图」拿到 `/uploads/xxx.jpg` 路径，复制粘贴到这里（图片本身可以最终不挂在任何商品上）。

**关于在线询价**：M3 阶段联系页只展示联系方式，访客自己拨电话/加微信。在线询价表单（弹窗、提交回邮件）是 M4 的功能。

## 4. 公开站玩法对照表

每次后台操作完，对照下面表格刷一下公开站验证：

| 后台操作 | 验证页面 | 看什么 |
|---|---|---|
| 新建 AVAILABLE 商品 | `/zh/products` | 第一页应出现 |
| 设置原价 | `/zh/products` 卡片 | 价格旁有划线原价 |
| 改成 SOLD | `/zh/products` | 已消失 |
| 新增分类 | `/zh/products` 筛选侧栏 | 「分类」下拉里出现 |
| 上传 CERT 附件 | `/zh/products/<slug>` 详情底部 | 出现「鉴定附件 (N)」折叠区 |
| 改 settings | `/zh/contact` | 电话/微信号/二维码同步 |
| 切英文站 | `/en/products` | 标题描述按英文显示，没填英文的回退中文 |

公开站布局要点：

- **首页** `/zh` — Hero + 分类入口 + 最新上架横滚
- **列表** `/zh/products` — 桌面带筛选侧栏（分类/品牌/价格区间），移动端是抽屉触发
- **详情** `/zh/products/<slug>` — 图轮播 + 信息 + 询价 CTA（链接到联系页）+ 收藏 + 鉴定附件折叠
- **收藏** `/zh/favorites` — 纯客户端 localStorage，清浏览器就丢了，访客无账号

## 5. 多语言策略

整个网站对照逻辑：

- **界面文案**（按钮、标签、提示）：由 `messages/zh.json` 和 `messages/en.json` 控制，店主**不要碰**，要改找技术。
- **商品标题/描述**：店主在后台双 Tab 填。英文留空 → 切到 `/en` 时回退到中文（不会变成空白）。
- **商品其他字段**（品牌、价格、编号、尺寸）：单语，所有语种共用。
- **分类名**：两种语种独立填，强烈建议都填。

> **推荐做法**：精品（爱马仕 Birkin、百达翡丽这种）认真翻一遍英文，能带来海外询单；普通货中文为主，英文留空也无伤大雅。

## 6. 安全建议

- **密码**：≥12 位，字母数字符号混排。`/admin/users` 改自己密码或新增同事账号。
- **同事账号**：店主用 OWNER，助手用 ADMIN。两者**唯一区别**是 ADMIN 不能管理其他管理员。给员工 ADMIN 即可。
- **离职**：员工走人立刻去 `/admin/users` 删除其账号，会话立即失效。
- **登录失败 5 次**会锁定 15 分钟（按邮箱限流），不会让人暴力破解。
- **截图分享后台**：注意不要把仪表盘上的询价联系方式（手机号/微信号）一起截图发到朋友圈。

## 7. 故障排查

| 现象 | 第一反应 | 下一步 |
|---|---|---|
| 新建商品后公开站看不到 | **状态是不是还停在 DRAFT？** | 列表里状态下拉改成 AVAILABLE |
| 公开站图片裂图 | URL 是不是真存在？外链域名是不是允许？ | 重新上传走 `/uploads/`，或让技术加图床域名到白名单 |
| 改了 settings 联系页不变 | 浏览器缓存了 | 强制刷新 Cmd+Shift+R |
| 一登录就 401 | session 失效或 JWT 配置变了 | 重新登录；持续失败找技术 |
| 上传图片报错 | 单图 > 5MB 或张数 > 15 | 压缩到 5MB 以下；单商品最多 15 张 |
| 公开站慢 | 图太大 + 没走 CDN | 上传前压缩到 200KB-1MB 即可；正式上线会接 COS+CDN |

## 9. 询价管理（M4）

### 9.1 访客怎么提交

- **商品详情页** → 点「询价」按钮 → 弹窗自动带上商品标题 → 填称呼/联系方式/留言 → 提交
- **联系页** → 点「提交咨询」按钮 → 弹窗（不带商品）→ 提交

服务端会校验字段长度、按 IP 限流（5 分钟最多 3 次）。

### 9.2 后台处理

`/admin/inquiries` 列表：

| 字段 | 说明 |
|---|---|
| 时间 | UTC 简短时间（年-月-日 时:分） |
| 访客 | 称呼 + 联系方式（电话或微信号）+ 提交语言 |
| 商品 | 有则点击新窗口打开详情页；无则显示「站点级」 |
| 留言 | 原文，保留换行 |
| 状态 | 未读 / 已读 / 已回复 |
| 操作 | "标记已读"（仅未读时显示）/ "删除" |

**未读筛选**：顶部下拉切到「未读」，只看待处理的。

**删除**是物理删除，无法撤销——除非需要清理脏数据，平时不要删（已读即可）。

### 9.3 隐私

- 访客的真实 IP **不入库**，只存 sha256 截 16 hex 作为限流键
- 联系方式（手机号/微信号）以明文存储，仅后台可见——截图分享请避开
